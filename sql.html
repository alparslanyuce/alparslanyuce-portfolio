Stock Forecast Function

Problem: Stok tahmini hatalı sipariş riskine yol açıyor.

Solution: Forecast function ile aylık planlama kontrolü yapıldı.

Impact: Fazla sipariş ve stok maliyeti azaltıldı.
Show Full SQL
CREATE FUNCTION dbo.EBFNC_SIPFORECAST1  
(  
    @STOK_KODU VARCHAR(50),  
    @TARIH DATE,  
    @MIKTAR DECIMAL(18,2)  
)  
RETURNS INT  
AS  
BEGIN  

    DECLARE @SONUC INT = 0  
    DECLARE @AY INT  
    DECLARE @YIL INT  

    DECLARE @FORECAST_MIKTAR DECIMAL(18,2) = 0  
    DECLARE @SIP_BAKIYE DECIMAL(18,2) = 0  

    SET @AY = MONTH(@TARIH)  
    SET @YIL = YEAR(@TARIH)  

    ;WITH FORECAST AS  
    (  
        SELECT   
            t.STOK_KODU,  
            v.AyNumarasi,  
            CASE   
                WHEN v.AyNumarasi < MONTH(GETDATE())   
                    THEN YEAR(GETDATE()) + 1  
                ELSE YEAR(GETDATE())  
            END AS YIL,  
            v.Miktar  
        FROM TBLSIP12TMP t  
        CROSS APPLY  
        (  
            VALUES  
                (1 , t.MIKTAR_1),  
                (2 , t.MIKTAR_2),  
                (3 , t.MIKTAR_3),  
                (4 , t.MIKTAR_4),  
                (5 , t.MIKTAR_5),  
                (6 , t.MIKTAR_6),  
                (7 , t.MIKTAR_7),  
                (8 , t.MIKTAR_8),  
                (9 , t.MIKTAR_9),  
                (10, t.MIKTAR_10),  
                (11, t.MIKTAR_11),  
                (12, t.MIKTAR_12)  
        ) v(AyNumarasi, Miktar)  
        WHERE t.STOK_KODU = @STOK_KODU  
    )  

    SELECT @FORECAST_MIKTAR = ISNULL(SUM(Miktar),0)
    FROM FORECAST
    WHERE AyNumarasi = @AY
    AND YIL = @YIL;

    IF (@SIP_BAKIYE + @MIKTAR) > @FORECAST_MIKTAR  
        RETURN 1;

    RETURN 0;

END
MRP ANALYSIS

Lead Time Analysis

Problem: Sipariş ve teslim süreçleri net takip edilemiyor.

Solution: Tarihler karşılaştırılarak gecikme analizi yapıldı.

Impact: Tedarik görünürlüğü artırıldı.
Show Full SQL
SELECT 
    t.STOK_KODU,
    t.TARIH,
    t.TESTAR,
    t.HABERTAR,
    t.SIPTAR,

    DATEDIFF(DAY, t.SIPTAR, t.TARIH) AS siparis_ihtiyac_fark,
    DATEDIFF(DAY, t.TESTAR, t.TARIH) AS teslim_ihtiyac_fark,
    DATEDIFF(DAY, t.HABERTAR, t.TESTAR) AS bildirim_teslim_fark,

    s.TEMIN_SURESI,
    c.NAKLIYE_SURESI

FROM TBLMRPSIP t
LEFT JOIN TBLSTSABIT s ON s.STOK_KODU = t.STOK_KODU
LEFT JOIN TBLCARISTOK c ON c.CARI_KOD = t.SATICI_KODU;
INVENTORY CONTROL

Stock Movement Analysis

Problem: Depo giriş/çıkış net görünmüyor.

Solution: Giriş/çıkış net stok hesaplandı.

Impact: Gerçek stok görünürlüğü sağlandı.
Show Full SQL
SELECT
    STOK_KODU,
    DEPO_KODU,
    SUM(CASE WHEN STHAR_GCKOD = 'G' THEN STHAR_GCMIK ELSE 0 END) AS GIRIS_MIKTAR,
    SUM(CASE WHEN STHAR_GCKOD = 'C' THEN STHAR_GCMIK ELSE 0 END) AS CIKIS_MIKTAR,
    SUM(
        CASE 
            WHEN STHAR_GCKOD = 'G' THEN STHAR_GCMIK 
            WHEN STHAR_GCKOD = 'C' THEN -STHAR_GCMIK 
            ELSE 0 
        END
    ) AS NET_MIKTAR
FROM TBLSTHAR
GROUP BY STOK_KODU, DEPO_KODU;
INVOICE TRACKING

Stock Invoice Analysis

Problem: Fatura bazlı stok hareketleri takip edilemiyor.

Solution: Stok ve fatura tabloları birleştirildi.

Impact: Finansal stok analizi sağlandı.
Show Full SQL
SELECT
F.SUBE_KODU,
F.TARIH,
F.FATIRS_NO AS BELGE_NO,
F.CARI_KODU,

CASE 
WHEN F.FTIRSIP = '1' THEN 'SATIS FATURASI'
WHEN F.FTIRSIP = '2' THEN 'ALIS FATURASI'
WHEN F.FTIRSIP = '3' THEN 'SATIS IRSALIYE'
WHEN F.FTIRSIP = '4' THEN 'ALIS IRSALIYE'
WHEN F.FTIRSIP IN ('8','9') THEN 'DEPOLAR ARASI TRANSFER'
END AS BELGE_TIPI,

S.STOK_KODU,
S.STHAR_GCMIK,
S.STHAR_NF

FROM TBLSTHAR S
LEFT JOIN TBLFATUIRS F ON F.FATIRS_NO = S.FISNO;
SERIAL STOCK

Serial / Non-Serial Merge

Problem: Serili ve serisiz stoklar ayrı takip ediliyor.

Solution: UNION ALL ile birleşik stok görünümü oluşturuldu.

Impact: Depo yönetimi kolaylaştırıldı.
Show Full SQL
-- SERİSİZ
SELECT
    sh.depo_kodu,
    sh.stok_kodu,
    sk.stok_adi,
    NULL AS seri_no,
    SUM(
        CASE 
            WHEN sh.sthar_gckod = 'G' THEN sh.sthar_gcmik
            WHEN sh.sthar_gckod = 'C' THEN -sh.sthar_gcmik
            ELSE 0
        END
    ) AS miktar
FROM tblsthar sh
JOIN TBLSTSABIT sk ON sk.stok_kodu = sh.stok_kodu
GROUP BY sh.depo_kodu, sh.stok_kodu, sk.stok_adi

UNION ALL

-- SERİLİ
SELECT
    ss.depokod,
    ss.stok_kodu,
    sk.stok_adi,
    ss.seri_no,
    SUM(
        CASE 
            WHEN ss.gckod = 'G' THEN ss.miktar
            WHEN ss.gckod = 'C' THEN -ss.miktar
            ELSE 0
        END
    ) AS miktar
FROM tblseritra ss
JOIN TBLSTSABIT sk ON sk.stok_kodu = ss.stok_kodu
GROUP BY ss.depokod, ss.stok_kodu, sk.stok_adi, ss.seri_no;
